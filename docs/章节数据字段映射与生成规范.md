### 第3章：数据总览（2-4页）【重构版 2026-06-12】

**核心变更：** 报备/非报备笔记区分改为根据业务底表「内容形式」字段判定，不再根据蒲公英是否拉取到数据判断。

**报备/非报备判定规则：**
```
报备笔记 = note_base.cooperation_form IN ('视频报备', '图文报备')
非报备笔记 = note_base.cooperation_form IN ('视频软文', '图文软文')
同义词：报备 = 官方合作 / 水上；非报备 = 非官方合作 / 水下
```

**业务底表字段（最终版）：**
```
发布链接（必须是长链接）| 内容形式 | 内容方向 | 笔记类型 | 资源含税成本价 | 资源含税售价 |
曝光量 | 阅读量 | 点赞量 | 收藏量 | 评论量 | 分享量 | 关注量 | 互动量
```

**数据源分工：**

| 类别 | 数据指标来源 | 费用来源 |
|------|-------------|---------|
| 报备笔记 | 蒲公英API (notes表): imp_num, read_num, like_num, fav_num, cmt_num, share_num | IF(内容金额口径='资源含税成本价', kol_price+total_platform_price, note_base.content_settlement) |
| 非报备笔记 | 业务底表 (note_base.metrics): impNum, readNum, likeNum, favNum, cmtNum, shareNum, followNum | IF(内容金额口径='资源含税成本价', note_base.content_cost, note_base.content_settlement) |

**合并计算公式（核心）：**
```
合并消费 = 报备消费 + 非报备消费
合并发布数量 = 报备发布数 + 非报备发布数
合并曝光量 = 报备曝光量 + 非报备曝光量
合并阅读量 = 报备阅读量 + 非报备阅读量
合并互动量 = 报备互动量 + 非报备互动量
合并爆文数 = 报备爆文数 + 非报备爆文数

整体CPM = 合并消费 / 合并曝光量 × 1000
整体CPC = 合并消费 / 合并阅读量
整体CPE = 合并消费 / 合并互动量
整体CTR = 合并阅读量 / 合并曝光量 × 100%
整体爆文率 = 合并爆文数 / 合并发布数量 × 100%

注：整体值 = 合并分子 ÷ 合并分母，非分别计算后加权平均
```

**互动量口径说明：**
```
报备互动：
  IF(互动口径='含关注', engage_num, like_num+fav_num+cmt_num+share_num) -- 蒲公英数据

非报备互动：
  IF(互动口径='含关注', 点赞量+收藏量+评论量+分享量+关注量, 点赞量+收藏量+评论量+分享量) -- 业务底表
```

**爆文判断：**
```
报备爆文：IF(爆文口径='转评赞', like_num+cmt_num+fav_num≥阈值, like_num≥阈值) -- 蒲公英数据
非报备爆文：IF(爆文口径='转评赞', 点赞量+收藏量+评论量≥阈值, 点赞量≥阈值) -- 业务底表
```

**自然流指标（仅对报备笔记有意义）：**
```
自然流曝光 = MAX(0, 报备曝光 - 聚光曝光)
自然流阅读 = MAX(0, 报备阅读 - 聚光点击)
自然流互动 = MAX(0, 报备互动 - 聚光互动)
```

**系统实现：** `src/pipeline/loaders/chapter-03-data-overview.ts` → `DataOverviewDataLoader`

**Prompt模板：** `src/prompts/chapters/chapter-03-data-overview.md`

**笔记级指标计算公式：**
```

笔记CPC = (kolPrice + serviceFee + fee) / read_num
笔记CPE = (kolPrice + serviceFee + fee) / IF(口径=含关注, like+fav+cmt+share+follow, like+fav+cmt+share)
```

### 第6章：内容分析/笔记侧（4-8页）【重构版 2026-06-12】

**核心变更：** 与数据总览相同，报备/非报备区分基于业务底表「内容形式」字段。每个维度内合并报备+非报备的数据再计算效率指标。

**分析维度：**
1. 内容方向分析（按 note_base.content_direction GROUP BY）
2. 内容形式分析（视频报备/视频软文 → 视频，图文报备/图文软文 → 图文）
3. 笔记类型分析（按 note_base.kol_type GROUP BY）
4. 达人层级分析（按 notes.kol_fan_num + review_configs.influencer_tiers 划分层级）
5. 优质笔记TOP5

**每组标准输出列：** 维度 | 消费 | 发布数量 | 爆文数 | 爆文率 | 曝光量 | 阅读量 | 互动量 | CTR | CPM | CPC | CPE

**每个维度的合并逻辑（以内容方向为例）：**
```
按内容方向分组后，每组内：
  合并消费 = 同维度报备消费 + 同维度非报备消费
  合并发布数 = 同维度报备发布数 + 同维度非报备发布数
  合并曝光量 = 同维度报备曝光量 + 同维度非报备曝光量
  合并阅读量 = 同维度报备阅读量 + 同维度非报备阅读量
  合并互动量 = 同维度报备互动量 + 同维度非报备互动量
  合并爆文数 = 同维度报备爆文数 + 同维度非报备爆文数
  该维度CPM = 合并消费 / 合并曝光量 × 1000
  该维度CPC = 合并消费 / 合并阅读量
  该维度CPE = 合并消费 / 合并互动量
  该维度CTR = 合并阅读量 / 合并曝光量 × 100%
  该维度爆文率 = 合并爆文数 / 合并发布数 × 100%
```

**达人层级说明：**
```
达人层级是派生字段，无原生数据；
根据蒲公英 kol_fan_num（达人粉丝数）+ 复盘配置中用户自定义阈值、分类名称划分层级。
层级数量不固定，先计算层级再分组统计。
```

**报备 vs 非报备数据源分工（同第3章）：**
- 报备笔记数据指标：蒲公英API (notes表)
- 非报备笔记数据指标：业务底表 (note_base.metrics JSON)
- 费用：按「内容金额口径」配置选择
