# 复盘报告数据溯源文档

> 本文档按章节分解复盘报告中所有数据和AI生成文本的来源，包括原始数据的数据库表、计算公式、以及AI文本生成的输入数据和提示词。

---

## 总体架构

```
报告生成流水线 (src/report/pipeline.ts)
├─ Step 1: 计算引擎 (src/calculation/pipeline.ts)
│   ├─ 读取: notes, juguang_data, kpi_targets, business_annotations
│   ├─ 计算: 15+ 指标函数
│   └─ 写入: calculated_metrics 表
├─ Step 2: 报告组装 (src/report/assembler.ts)
│   ├─ 读取: 所有项目关联数据
│   └─ 组装: 12个固定模块
├─ Step 3: AI文本生成 (src/report/ai-service.ts)
│   ├─ 输入: AllMetrics + Highlights
│   ├─ 调用: LLM (OpenAI)
│   └─ 保存: ai_generated_content 表
└─ Step 4: 导出 (src/report/exporter.ts)
    └─ 格式: JSON/PDF/HTML
```

## 第3章：数据总览 (data_overview)

### 数据溯源

| 展示字段 | 原始数据来源 | 计算公式 | 代码位置 |
|---------|------------|---------|---------|
| 总笔记篇数 | notes表 | `COUNT(notes WHERE project_id=X)` | assembler.ts |
| **总费用** | notes + juguang_data | 见下方公式 | src/calculation/cost.ts |
| 水上费用 | notes表 (kolPrice, serviceFee) | `SUM((kolPrice + serviceFee) × discount)` | calculateProjectTotalCost() |
| 水下费用 | notes表 (underwaterPrice) | `SUM(underwaterPrice)` (不应用折扣) | calculateProjectTotalCost() |
| 聚光费用 | juguang_data表 (fee) | `SUM(fee)` | calculateProjectTotalCost() |
| 总费用 | 计算得出 | `aboveWaterCost + underwaterCost + juguangCost` | calculateProjectTotalCost() |
| **总曝光量** | notes表 (imp_num) | `SUM(impNum)` | pipeline.ts |
| **总阅读量** | notes表 (read_num) | `SUM(readNum)` | pipeline.ts |
| **总互动量** | notes表 | `SUM(likeNum + favNum + cmtNum [+ shareNum] [+ followNum])` | calculateEngagement() |
| **CPM** | 计算得出 | `totalCost / totalImpressions × 1000` | src/calculation/metrics.ts |
| **CPC** | 计算得出 | `totalCost / totalReads` | src/calculation/metrics.ts |
| **CPE** | 计算得出 | `totalCost / totalEngagement` | src/calculation/metrics.ts |
| **CTR** | 计算得出 | `totalReads / totalImpressions` | src/calculation/metrics.ts |
| **爆文数** | notes表 | `COUNT(notes WHERE likeNum+favNum+cmtNum >= 1000)` | src/calculation/viral.ts |
| **爆文率** | 计算得出 | `viralCount / totalNoteCount` | calculateViralRate() |
| KPI目标值 | kpi_targets表 | `WHERE project_id AND metric_name` → targetValue | 直接读取 |
| **KPI完成率** | 计算得出 | 非成本类: `actual / target`; 成本类: `target / actual` | src/calculation/kpi.ts |
| 大盘基准值 | manual_inputs / review_configs | inputType='benchmark' → {ctr,cpm,cpc,cpe} | 直接读取 |
| **大盘对比** | 计算得出 | 非成本类: `(actual-benchmark)/benchmark×100`; 成本类: `(benchmark-actual)/benchmark×100` | src/calculation/benchmark.ts |

### 计算公式详解

#### 互动量计算 (可配置口径)
```
engagement = likeNum + favNum + cmtNum
if (engagementConfig.includeShare) engagement += shareNum
if (engagementConfig.includeFollow) engagement += followNum
```
- 配置来源: `projects.engagement_config` 或 `review_configs.engagementMetric`

#### 爆文判定 (固定口径，不受配置影响)
```
isViral = (likeNum + favNum + cmtNum) >= 1000
```

#### 费用折扣逻辑
```
discount = specialRules[kolId]?.discount ?? cooperationPolicy.defaultDiscount
aboveWaterCost = SUM((kolPrice + serviceFee) × discount)
```
- 配置来源: `projects.cooperation_policy` → {defaultDiscount, specialRules[]}

## 第6章：内容分析 (content_analysis)

### 数据溯源

| 展示字段 | 数据库表 | 数据库列 | 聚合方式 |
|---------|---------|---------|---------|
| 按内容方向分组 | business_annotations | content_direction | GROUP BY content_direction |
| 按笔记类型分组 | notes | note_type | GROUP BY note_type |
| 按达人层级分组 | notes (kolFanNum) | kol_fan_num | classifyKOLTier() 后 GROUP BY tier |
| 按投放阶段分组 | business_annotations | launch_phase | GROUP BY launch_phase |
| 各分组指标 | notes + juguang_data | 多列 | SUM/AVG per group |

### 达人层级分类规则
```
classifyKOLTier(fanCount):
  头部KOL: fanCount >= 头部阈值 (来自 review_configs.influencerTiers)
  腰部KOL: fanCount >= 腰部阈值
  尾部KOL: fanCount >= 尾部阈值
  KOC: 其余
```
- 配置来源: `review_configs.influencer_tiers` (JSON数组)

### 维度聚合计算
```
aggregateByDimension(notes, annotations, dimension):
  对每个维度值:
    noteCount = COUNT(notes)
    totalImp = SUM(impNum)
    totalRead = SUM(readNum)
    totalEng = SUM(engagement)
    viralCount = COUNT(viral notes)
    viralRate = viralCount / noteCount
    avgCPE = totalCost / totalEng
```
- **代码位置**: `src/calculation/content.ts` → `aggregateByDimension()`

## 附录C：AI生成文本溯源链路

每段AI生成文本的完整溯源链路：

```
AI生成文本
├─ 提示词模板 (src/prompts/chapters/chapter-XX-*.md)
│   └─ 包含: 系统角色设定 + 输出格式要求
├─ 输入数据 (来自 calculated_metrics 或直接查询)

│   └─ 配置数据: review_configs/kpi_targets 表
├─ LLM调用参数
│   ├─ model: 环境变量 OPENAI_MODEL (默认 gpt-4)
│   ├─ temperature: 0.3(解析类) / 0.7(生成类)
│   └─ max_tokens: 按章节配置
└─ 输出存储: ai_generated_content 表
    ├─ content_type: 标识章节类型
    ├─ generated_content: AI原始输出
    ├─ edited_content: 人工编辑版本
    └─ is_edited: 是否经过人工修改
```

## 附录D：配置对计算的影响

| 配置项 | 来源 | 影响范围 | 说明 |
|--------|------|---------|------|
| engagementConfig | projects.engagement_config | 互动量/CPE/KPI完成率 | 决定互动量是否包含分享和关注 |
| viralMetric | review_configs | 爆文判定/爆文率 | 固定阈值1000(点赞+收藏+评论) |
| influencerTiers | review_configs | 达人层级分类 | 自定义粉丝量阈值 |
| cooperationPolicy | projects | 费用计算 | 折扣规则影响水上费用 |
| benchmark | review_configs/manual_inputs | 大盘对比 | 行业基准值 |
| modules | review_configs | 章节显示/隐藏 | 控制哪些模块出现在报告中 |

当 `engagementConfig` 变更时，系统会触发 `onEngagementConfigChange()` 重新执行完整计算流水线。
