# 复盘报告各章节生成数据与提示词规范

> 本文档按报告章节顺序，逐章整理：(1) 该章节需要的全部数据及来源 (2) 生成该章节内容的提示词模板

---

## 第1章：封面

### 数据需求

| 数据项 | 来源 | 取值方式 |
|--------|------|---------|
| 品类 | projects表 | `project.category` |
| 合作品牌 | projects表 | `project.brand` |
| 品牌业务线 | projects表 | `project.businessLine` |
| 项目名称 | projects表 | `project.projectName` |
| 项目执行周期 | projects表 | `project.startDate` ~ `project.endDate` |

### 生成提示词

```
你是一位专业的小红书营销复盘报告设计师。请为以下项目生成封面内容：

## 项目信息
- 品类：{{category}}
- 品牌：{{brand}}
- 业务线：{{businessLine}}
- 项目名称：{{projectName}}
- 执行周期：{{startDate}} — {{endDate}}

## 输出要求
生成封面展示文案，格式为：
1. 主标题：品牌名 + 项目名称
2. 副标题："小红书种草项目复盘报告"
3. 时间标注：执行周期
4. 出品方标注（如有）

输出纯文本，不需要排版指令。
```

## 第2章：项目回顾

### 数据需求

| 数据项 | 来源 | 取值方式 |
|--------|------|---------|
| 项目背景（客户期待） | AI解析策划案 | `ai_generated_content` WHERE content_type='plan_parse' → generatedContent.projectObjective |
| 传播目的 | AI解析策划案 | 同上 → generatedContent.strategy |
| 策略回顾 | AI解析策划案 | 同上 → generatedContent.targetAudience + coreMessage |
| 传播节奏/阶段 | 复盘配置 | `review_configs.launchPhases` (Json数组: [{name, startDate, endDate, noteCount}]) |
| 各阶段发布篇数 | notes表+业务标注 | GROUP BY launch_phase, COUNT(note_id) |

### 生成提示词

```
你是一位资深的小红书营销复盘专家。请基于以下策划案解析结果，生成"项目回顾"章节内容。

## 策划案解析数据
- 项目背景/客户期待：{{projectObjective}}
- 传播目的：{{strategy}}
- 目标人群：{{targetAudience}}
- 核心传播信息：{{coreMessage}}

## 传播节奏
{{#each launchPhases}}
- {{name}}：{{startDate}} ~ {{endDate}}，发布{{noteCount}}篇
{{/each}}

## 输出要求
请输出以下三个部分：

### 1. 项目背景（80-150字）
概述品牌诉求和项目目标，突出客户核心期待。若策划案中未找到相关内容，标注「未找到」。

### 2. 传播目的（50-100字）
提炼核心传播目标，可分维度列出（如品牌价值/产品声量/活动造势）。

### 3. 策略回顾（100-200字）
按时间轴梳理各阶段策略要点，说明每阶段的核心动作和发布节奏。

要求：语言专业简洁，使用营销行业术语，避免口语化表达。
```

## 第3章：数据总览

### 数据需求

| 数据项 | 来源 | 计算公式 |
|--------|------|---------|
| 总笔记篇数 | notes表 | COUNT(notes WHERE project_id) |
| 总费用 | 计算引擎 | aboveWaterCost + underwaterCost + juguangCost |
| 达人金额 | notes表 | SUM(kolPrice + serviceFee) (水上) + SUM(underwaterPrice) (水下) |
| 投流金额 | juguang_data表 | SUM(fee) |
| KPI-总曝光 | kpi_targets表 | WHERE metric_name='impression' → targetValue |
| 实际达成-总曝光 | notes表 | SUM(impNum) |
| 完成率-总曝光 | 计算 | SUM(impNum) / KPI * 100 |
| KPI-总阅读 | kpi_targets表 | WHERE metric_name='read' → targetValue |
| 实际达成-总阅读 | notes表 | SUM(readNum) |
| 完成率-总阅读 | 计算 | SUM(readNum) / KPI * 100 |
| KPI-总互动 | kpi_targets表 | WHERE metric_name='engagement' → targetValue |
| 实际达成-总互动 | notes表+配置 | IF(includeFollow, SUM(like+fav+cmt+share+follow), SUM(like+fav+cmt+share)) |
| 完成率-总互动 | 计算 | 实际互动 / KPI * 100 |
| KPI-爆文率 | kpi_targets表 | WHERE metric_name='viralRate' → targetValue |
| 实际达成-爆文数 | notes表 | COUNT WHERE (like+fav+cmt >= 1000) |
| 实际达成-爆文率 | 计算 | 爆文数 / 总笔记数 * 100 |
| 完成率-爆文率 | 计算 | 实际爆文率 / KPI爆文率 * 100 |
| KPI-CPM | kpi_targets表 | WHERE metric_name='cpm' → targetValue |
| 实际达成-CPM | 计算 | 总费用 / SUM(impNum) * 1000 |
| 完成率-CPM | 计算 | KPI / 实际CPM * 100 (成本指标反转) |
| KPI-CPC | kpi_targets表 | WHERE metric_name='cpc' → targetValue |
| 实际达成-CPC | 计算 | 总费用 / SUM(readNum) |
| 完成率-CPC | 计算 | KPI / 实际CPC * 100 |
| KPI-CPE | kpi_targets表 | WHERE metric_name='cpe' → targetValue |
| 实际达成-CPE | 计算 | 总费用 / 总互动 |
| 完成率-CPE | 计算 | KPI / 实际CPE * 100 |
| KPI-CTR | kpi_targets表 | WHERE metric_name='ctr' → targetValue |
| 实际达成-CTR | 计算 | SUM(readNum) / SUM(impNum) * 100 |
| 完成率-CTR | 计算 | 实际CTR / KPI * 100 |
| KPI-搜索指数 | kpi_targets表 | WHERE metric_name='search_index' (非必展示) |
| 实际达成-搜索指数 | lingxi_data表 | WHERE data_type='search_index' → dataContent.value |
| KPI-品牌AIPS | kpi_targets表 | WHERE metric_name='brand_aips' (非必展示) |
| 实际达成-品牌AIPS | lingxi_data表 | WHERE data_type='aips' → dataContent.awareness |
| KPI-TI人群 | kpi_targets表 | WHERE metric_name='brand_ti' (非必展示) |
| 实际达成-TI人群 | juguang_data表 | SUM(tiUserNum) |
| 自然流曝光量 | 计算 | SUM(impNum) - SUM(impression) |
| 自然流阅读量 | 计算 | SUM(readNum) - SUM(click) |
| 自然流互动量 | 计算 | 总互动 - SUM(interaction) |
| 自然流CPM | 计算 | 总费用 / 自然流曝光 * 1000 |
| 自然流CPC | 计算 | 总费用 / 自然流阅读 |
| 自然流CPE | 计算 | 总费用 / 自然流互动 |
| 自然流CTR | 计算 | 自然流阅读 / 自然流曝光 * 100 |

### 生成提示词

```
你是一位资深的小红书营销数据分析师。请基于以下数据总览信息，生成数据解读文案。

## 项目概况

- 总笔记：{{noteCount}}篇
- 总费用：{{totalCost}}元（达人{{aboveWaterCost}}元 + 投流{{juguangCost}}元）

## KPI完成情况
| 指标 | KPI目标 | 实际达成 | 完成率 |
|------|--------|---------|--------|
| 总曝光 | {{kpi_impression}}万 | {{actual_impression}}万 | {{rate_impression}}% |
| 总阅读 | {{kpi_read}}万 | {{actual_read}}万 | {{rate_read}}% |
| 总互动 | {{kpi_engagement}}万 | {{actual_engagement}}万 | {{rate_engagement}}% |
| 爆文率 | {{kpi_viralRate}}% | {{actual_viralRate}}% | {{rate_viralRate}}% |
| CPM | {{kpi_cpm}} | {{actual_cpm}} | {{rate_cpm}}% |
| CPC | {{kpi_cpc}} | {{actual_cpc}} | {{rate_cpc}}% |
| CPE | {{kpi_cpe}} | {{actual_cpe}} | {{rate_cpe}}% |

## 自然流量数据
- 自然流曝光：{{natural_impression}}万
- 自然流阅读：{{natural_read}}万
- 自然流互动：{{natural_engagement}}万
- 自然流CPM：{{natural_cpm}}
- 自然流CTR：{{natural_ctr}}%

## 输出要求
请生成数据总览解读（150-250字），包含：
1. 整体完成情况一句话总结（如"各项KPI均超额完成"或"核心指标达标，部分指标待优化"）
2. 亮点指标分析（完成率>100%的指标，说明超额原因）

语气要求：专业客观，数据驱动，避免过度修饰。所有结论必须有数据支撑。
```

## 第4章：项目亮点

### 数据需求

| 数据项 | 来源 | 计算公式 |
|--------|------|---------|
| KPI超额完成列表 | 计算引擎 | identifyHighlights() → type='kpi_exceeded' |
| 优于大盘列表 | 计算引擎 | identifyHighlights() → type='above_benchmark' |
| 大盘基准值 | manual_inputs/review_configs | benchmark: {ctr, cpm, cpc, cpe} |
| 实际CPM vs 大盘CPM | 计算 | (大盘CPM - 实际CPM) / 大盘CPM * 100 |

| 品牌搜索排名 | lingxi_data表 | WHERE data_type='brand_ranking' → {rank, category} |
| AIPS人群数据 | lingxi_data表 | WHERE data_type='aips' → {awareness, interest, purchase, share} |

## 第5章：综合分析（四象限）

### 数据需求

| 数据项 | 来源 | 计算公式 |
|--------|------|---------|
| 笔记级CPM | 计算 | (kolPrice+serviceFee+fee) / impNum * 1000 (每条笔记) |
| 笔记级CPC | 计算 | (kolPrice+serviceFee+fee) / readNum |
| 笔记级CPE | 计算 | (kolPrice+serviceFee+fee) / 互动量(按口径) |
| 笔记级CTR | 计算 | readNum / impNum |
| 投流级CPM | 计算 | fee / impression * 1000 (每条笔记的聚光数据) |
| 投流级CPC | 计算 | fee / click |
| 投流级CPE | 计算 | fee / interaction |
| 投流级CTR | 计算 | click / impression |
| 笔记质量得分 | 计算 | 选中指标在项目内排名归一化(0-100) |
| 投流质量得分 | 计算 | 选中指标在项目内排名归一化(0-100) |
| 项目均值(笔记侧) | 计算 | AVG(笔记质量得分) |
| 项目均值(投流侧) | 计算 | AVG(投流质量得分) |
| 象限归属 | 计算 | 笔记>=均值 AND 投流>=均值 → 核心资产... |
| 各象限笔记数 | 计算 | GROUP BY 象限, COUNT |
| 各象限总费用 | 计算 | GROUP BY 象限, SUM(kolPrice+serviceFee+fee) |
| 达人昵称 | notes表 | kolNickName |
| 内容方向 | business_annotations表 | contentDirection |
| 合作形式 | notes表 | noteType (1=图文, 2=视频) |

### 生成提示词

```
你是一位资深的小红书营销数据分析师。请基于以下四象限分析数据，生成综合分析解读。

## 当前分析指标：{{selectedMetric}}（{{metricDescription}}）
- X轴：投流侧{{selectedMetric}}原始值（越{{xDirection}}越好）
- Y轴：笔记侧{{selectedMetric}}原始值（越{{yDirection}}越好）
- 分割线：项目内均值（笔记侧均值={{noteAvg}}，投流侧均值={{trafficAvg}}）

## 四象限分布
| 象限 | 笔记数 | 总费用 | 占比 |
|------|--------|--------|------|
| 核心资产（笔记好+投流好） | {{coreCount}}条 | {{coreCost}}元 | {{corePercent}}% |
| 潜力内容（笔记好+投流差） | {{potentialCount}}条 | {{potentialCost}}元 | {{potentialPercent}}% |
| 流量消耗（笔记差+投流好） | {{trafficCount}}条 | {{trafficCost}}元 | {{trafficPercent}}% |
| 淘汰候选（笔记差+投流差） | {{eliminateCount}}条 | {{eliminateCost}}元 | {{eliminatePercent}}% |

## 各象限典型笔记
### 核心资产TOP3
{{#each coreTop3}}
- {{kolNickName}} | {{contentDirection}} | 笔记{{selectedMetric}}={{noteMetricValue}} | 投流{{selectedMetric}}={{trafficMetricValue}}
{{/each}}

### 淘汰候选TOP3
{{#each eliminateTop3}}
- {{kolNickName}} | {{contentDirection}} | 笔记{{selectedMetric}}={{noteMetricValue}} | 投流{{selectedMetric}}={{trafficMetricValue}}
{{/each}}

## 输出要求
请生成综合分析解读（200-350字），包含：

### 1. 指标说明（30字）
说明当前选中指标的含义：X轴代表什么，Y轴代表什么，均值基准的意义。

### 2. 分布概述（50字）
各象限数量和费用占比的整体描述。

### 3. 各象限分析（每象限50-80字）
- **核心资产**（{{coreCount}}条）：内容特征 + 达人特征 + 建议（持续投放/加大预算）
- **潜力内容**（{{potentialCount}}条）：内容特征 + 投流优化方向
- **流量消耗**（{{trafficCount}}条）：问题诊断 + 内容优化方向
- **淘汰候选**（{{eliminateCount}}条）：问题总结 + 替换建议

### 4. 行动建议（50字）
基于象限分布给出1-2条核心优化建议。

语气要求：数据驱动，结论明确，建议可操作。
```
