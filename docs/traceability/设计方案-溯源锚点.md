# 复盘报告溯源锚点设计方案

## 需求

在审校台中，AI生成的每个章节段落旁显示溯源锚点（图标按钮），点击后展示该段落对应的：
- 原始数据（数据库查询结果）
- 计算公式（派生指标的推导过程）
- AI输入（传给LLM的变量值）

---

│                                                             │

┌─────────────────────────────────────────────────────────────┐
│                    审校台前端                                 │

## 数据模型设计

### ChapterData 扩展

```typescript
// 现有结构
interface ChapterData {
  id: string;
  title: string;
  number: number;
  content: string;  // HTML
}

// 扩展后
interface ChapterData {
  id: string;
  title: string;
  number: number;
  content: string;  // HTML
  traceability?: ChapterTraceability;  // ★ 新增
}

interface ChapterTraceability {
  // 该章节使用的原始数据快照
  rawData: Record<string, RawDataSource>;
  // 计算过程
  calculations: CalculationTrace[];
  // 传给LLM的完整变量
  inputVariables: Record<string, string>;
  // 提示词模板路径
  promptTemplate: string;
  // 生成时间
  generatedAt: string;
}

interface RawDataSource {
  table: string;           // 数据库表名
  query: string;           // SQL查询（参数化）
  rowCount: number;        // 返回行数
  sample: unknown[];       // 数据样本（前5行）
  columns: string[];       // 列名列表
}

interface CalculationTrace {
  metric: string;          // 指标名称（如 "CPM"）
  formula: string;         // 公式文本（如 "totalCost / totalImpressions × 1000"）
  inputs: Record<string, number | string>;  // 输入值
  result: number | string; // 计算结果
}
```

## 实现方案（分3层）

### 第1层：DataLoader 收集溯源数据

修改 `BaseChapterDataLoader`，在 `ChapterDataContext` 中新增 `traceability` 字段：

```typescript
// src/pipeline/loaders/types.ts 扩展
export interface ChapterDataContext {
  variables: Record<string, string>;
  missingFields: string[];
  traceability?: ChapterTraceability;  // ★ 新增
}
```

每个 DataLoader 在查询数据时，同时记录：
- 执行的查询（表名 + 条件）
- 返回的原始数据样本
- 计算过程（公式 + 输入值 + 结果）

**示例（chapter-03-data-overview.ts）：**

```typescript
// 在 load() 方法中
const traceability: ChapterTraceability = {
  rawData: {},
  calculations: [],
  inputVariables: {},
  promptTemplate: 'src/prompts/chapters/chapter-03-data-overview.md',
  generatedAt: new Date().toISOString(),
};

// 查询笔记数据时记录
const notes = await this.prisma.note.findMany({ where: { projectId } });
traceability.rawData['notes'] = {
  table: 'notes',
  query: `SELECT * FROM notes WHERE project_id = '${projectId}'`,
  rowCount: notes.length,
  sample: notes.slice(0, 5),
  columns: ['note_id', 'imp_num', 'read_num', 'like_num', ...],
};

// 计算CPM时记录
const cpm = totalImpressions > 0 ? (totalCost / totalImpressions) * 1000 : 0;
traceability.calculations.push({
  metric: 'CPM',
  formula: 'totalCost / totalImpressions × 1000',
  inputs: { totalCost, totalImpressions },
  result: cpm,
});
```

### 第3层：前端审校台展示

#### 3.1 锚点按钮注入

在审校台渲染章节内容时，为每个段落/表格/图表旁添加溯源锚点：

```tsx
// web/src/components/proofread/traceability-anchor.tsx

interface TraceabilityAnchorProps {
  chapterTraceability: ChapterTraceability;
  onOpen: () => void;
}

function TraceabilityAnchor({ onOpen }: TraceabilityAnchorProps) {
  return (
    <button
      onClick={onOpen}
      className="traceability-anchor"
      title="查看数据来源"
    >
      <DatabaseIcon size={14} />
    </button>
  );
}
```

#### 3.2 溯源面板（侧边抽屉）

点击锚点后，在右侧展开抽屉面板，分Tab展示：

```tsx
// web/src/components/proofread/traceability-panel.tsx

function TraceabilityPanel({ traceability, onClose }) {
  return (
    <aside className="traceability-panel">
      <Tabs>
        <Tab label="原始数据">
          {/* 展示各表查询结果 */}
          {Object.entries(traceability.rawData).map(([key, source]) => (
            <DataSourceCard key={key} source={source} />
          ))}
        </Tab>

        <Tab label="计算过程">
          {/* 展示每个派生指标的公式和输入值 */}
          {traceability.calculations.map((calc) => (
            <CalculationCard key={calc.metric} calc={calc} />
          ))}
        </Tab>

        <Tab label="AI输入">
          {/* 展示传给LLM的变量值 */}
          <VariablesTable variables={traceability.inputVariables} />
          <p>提示词模板: {traceability.promptTemplate}</p>
        </Tab>
      </Tabs>
    </aside>
  );
}
```

#### 3.3 数据源卡片

```tsx
function DataSourceCard({ source }: { source: RawDataSource }) {
  return (
    <div className="data-source-card">
      <h4>{source.table} ({source.rowCount}行)</h4>
      <code className="sql-query">{source.query}</code>
      <table>
        <thead>
          <tr>{source.columns.map(col => <th key={col}>{col}</th>)}</tr>
        </thead>
        <tbody>
          {source.sample.map((row, i) => (
            <tr key={i}>
              {source.columns.map(col => <td key={col}>{row[col]}</td>)}
            </tr>
          ))}
        </tbody>
      </table>
      {source.rowCount > 5 && <p className="text-xs">仅展示前5行...</p>}
    </div>
  );
}
```

#### 3.4 计算过程卡片

```tsx
function CalculationCard({ calc }: { calc: CalculationTrace }) {
  return (
    <div className="calculation-card">
      <h4>{calc.metric}</h4>
      <div className="formula">{calc.formula}</div>
      <div className="inputs">
        {Object.entries(calc.inputs).map(([k, v]) => (
          <span key={k}>{k} = {v}</span>
        ))}
      </div>
      <div className="result">= {calc.result}</div>
    </div>
  );
}
```

## 锚点定位策略

### 方案A：章节级锚点（推荐先实现）

每个章节标题旁一个锚点，点击展示该章节的全部溯源数据。

**优点**：实现简单，不需要修改LLM输出格式
**缺点**：粒度较粗

### 方案B：段落级锚点（进阶）

在LLM输出的HTML中，通过 `data-trace-*` 属性标记每个段落对应的数据来源：

```html
<!-- LLM输出时添加标记 -->
<p data-trace-metrics="cpm,cpc,cpe">
  本次投放CPM为45.2元，优于大盘均值50元...
</p>
<table data-trace-source="notes" data-trace-group="by_content_direction">
  ...
</table>
```

前端解析这些属性，在对应元素旁渲染锚点按钮，点击后只展示相关的数据子集。

**实现方式**：在 `HTML_OUTPUT_INSTRUCTION` 中增加规则，要求LLM在输出HTML时添加 `data-trace-*` 属性。

## 实施路径（分阶段）

### Phase 1：章节级溯源（MVP）

1. 扩展 `ChapterDataContext` 类型，增加 `traceability` 字段
2. 修改各 DataLoader，在查询时收集原始数据和计算过程
3. Stream route 将 traceability 随 chapter 传递
4. 前端在章节标题旁添加锚点按钮
5. 点击后弹出侧边面板展示数据

**改动文件**：
- `src/pipeline/loaders/types.ts` — 类型扩展
- `src/pipeline/loaders/chapter-*.ts` — 各loader收集数据
- `web/src/app/api/generate-report/[reviewConfigId]/stream/route.ts` — 传递数据
- `web/src/app/review/[id]/proofread/page.tsx` — 渲染锚点
- `web/src/components/proofread/traceability-panel.tsx` — 新建面板组件

### Phase 2：段落级溯源

1. 修改 `HTML_OUTPUT_INSTRUCTION`，要求LLM输出 `data-trace-*` 属性
2. 前端解析 `data-trace-*` 属性，为每个标记元素渲染锚点
3. 锚点点击后只展示该段落相关的数据子集

### Phase 3：交互增强

1. 锚点hover预览（tooltip显示关键指标）
2. 数据变更高亮（标记哪些原始数据影响了该段落）
3. 支持从溯源面板跳转到数据库管理界面
4. 导出溯源报告（附带原始数据的完整审计文档）

## 数据量控制

溯源数据可能较大，需要控制存储量：

| 策略 | 说明 |
|------|------|
| 样本限制 | rawData.sample 最多5行 |
| 列过滤 | 只保留与该章节相关的列 |
| 懒加载 | 前端按需请求完整数据（API） |
| 分离存储 | traceability 单独存表，不放在 reportContent JSON 中 |

### 推荐：分离存储方案

```sql
-- 新建溯源数据表
CREATE TABLE report_traceability (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  review_config_id UUID NOT NULL REFERENCES review_configs(id),
  chapter_id VARCHAR(50) NOT NULL,
  chapter_number INT NOT NULL,
  raw_data JSONB,          -- 原始数据快照
  calculations JSONB,      -- 计算过程
  input_variables JSONB,   -- AI输入变量
  prompt_template VARCHAR(200),
  generated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(review_config_id, chapter_id)
);
```

前端通过 API 按需加载：

```
GET /api/reviews/{id}/traceability/{chapterId}
```

## 关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 锚点粒度 | Phase1 章节级，Phase2 段落级 | 渐进式实现，MVP快速上线 |
| 存储方式 | 分离表 report_traceability | 避免 reportContent JSON 过大 |
| 数据加载 | 前端按需请求 | 减少初始加载量 |
| 原始数据量 | 样本5行 + 总行数 | 平衡可读性和存储 |
| 计算展示 | 公式 + 输入值 + 结果 | 完整可验证 |
