# 段落级溯源锚点设计方案

## 需求

在审校台中，AI生成的每个**段落/表格/图表**旁显示溯源锚点，点击后展示该段落引用的原始数据。数据量大时分页显示，每页固定行数。

---

## 核心思路

1. LLM 输出 HTML 时，在每个数据段落上标记 `data-trace-id`（溯源标识）
2. DataLoader 按 trace-id 粒度存储对应的原始数据到独立表
3. 前端解析 `data-trace-id`，渲染锚点按钮
4. 点击锚点 -> 调用分页 API 获取该段落的原始数据
5. 弹出面板展示：原始数据表格（分页）+ 计算公式

---

## 数据流

```
生成阶段:
  DataLoader.load()
    |-- 查询DB -> 按段落粒度拆分数据
    |-- 计算指标 -> 记录公式+输入+结果
    +-- 输出 traceItems: TraceItem[]

  LLM生成HTML
    +-- 在 HTML_OUTPUT_INSTRUCTION 中要求输出 data-trace-id 属性

  保存阶段
    +-- traceItems 写入 report_trace_items 表

查看阶段:
  前端渲染章节HTML
    |-- 扫描 [data-trace-id] 元素
    |-- 每个元素旁注入锚点按钮
    +-- 点击 -> GET /api/reviews/{id}/trace/{traceId}?page=1&pageSize=10
              -> 返回该段落的原始数据（分页）+ 计算公式
```

## 数据模型

### 溯源条目表

```sql
CREATE TABLE report_trace_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  review_config_id UUID NOT NULL REFERENCES review_configs(id) ON DELETE CASCADE,
  trace_id VARCHAR(100) NOT NULL,       -- 段落标识，如 "ch3_kpi_table"
  chapter_number INT NOT NULL,
  label VARCHAR(200) NOT NULL,          -- 展示名称，如 "KPI达成数据"

  -- 原始数据
  source_table VARCHAR(100) NOT NULL,   -- 数据库表名
  source_query TEXT NOT NULL,           -- SQL查询（参数化）
  total_rows INT NOT NULL DEFAULT 0,    -- 总行数
  columns JSONB NOT NULL,               -- 列定义 [{key, label, type}]
  data_rows JSONB NOT NULL,             -- 全部数据行（JSON数组）

  -- 计算公式（可选，仅派生指标有）
  calculations JSONB,                   -- [{metric, formula, inputs, result}]

  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(review_config_id, trace_id)
);

CREATE INDEX idx_trace_items_review ON report_trace_items(review_config_id);
```

### TypeScript 类型

```typescript
interface TraceItem {
  traceId: string;          // 段落标识
  chapterNumber: number;
  label: string;            // "KPI达成数据"
  sourceTable: string;      // "notes"
  sourceQuery: string;      // "SELECT ... FROM notes WHERE ..."
  totalRows: number;
  columns: TraceColumn[];
  dataRows: Record<string, unknown>[];
  calculations?: CalculationTrace[];
}

interface TraceColumn {
  key: string;       // 字段名
  label: string;     // 中文列名
  type: 'string' | 'number' | 'date' | 'boolean';
}

interface CalculationTrace {
  metric: string;                       // "CPM"
  formula: string;                      // "totalCost / totalImpressions * 1000"
  inputs: Record<string, number>;       // {totalCost: 128000, totalImpressions: 6200000}
  result: number | string;              // 20.65
}
```

## LLM 输出规范

在 `HTML_OUTPUT_INSTRUCTION` 中新增规则：

```
9. 每个包含数据引用的段落、表格、列表，必须添加 data-trace-id 属性标识数据来源：
   - 数据表格: <table data-trace-id="ch3_kpi_table" class="report-table">
   - 数据段落: <p data-trace-id="ch3_cost_summary">总费用128,000元...</p>
   - 数据列表: <ul data-trace-id="ch4_highlights_list">
   data-trace-id 命名规则: ch{章节号}_{模块}_{类型}，如 ch3_kpi_table, ch7_traffic_overview
   纯文字分析段落（不直接引用数据）不需要添加此属性。
```

## 前端实现

### 锚点注入 Hook

```typescript
// web/src/hooks/use-trace-anchors.ts
export function useTraceAnchors(
  containerRef: RefObject<HTMLElement>,
  content: string,
  onAnchorClick: (traceId: string) => void
) {
  useEffect(() => {
    if (!containerRef.current) return;
    // 查找所有带 data-trace-id 的元素
    const traceElements = containerRef.current.querySelectorAll('[data-trace-id]');
    traceElements.forEach((el) => {
      const traceId = el.getAttribute('data-trace-id')!;
      if (el.querySelector('.trace-anchor-btn')) return; // 避免重复
      const btn = document.createElement('button');
      btn.className = 'trace-anchor-btn';
      btn.title = '查看数据来源';
      btn.innerHTML = '<svg>...</svg>'; // 数据库图标
      btn.onclick = (e) => { e.stopPropagation(); onAnchorClick(traceId); };
      el.style.position = 'relative';
      el.appendChild(btn);
    });
  }, [content]);
}
```

### 溯源面板（分页表格）

```tsx
// web/src/components/proofread/trace-data-panel.tsx
function TraceDataPanel({ reviewId, traceId, onClose }) {
  const [page, setPage] = useState(1);
  const pageSize = 10;

  const { data, isLoading } = useQuery({
    queryKey: ['trace', reviewId, traceId, page],
    queryFn: () => fetch(
      `/api/reviews/${reviewId}/trace/${traceId}?page=${page}&pageSize=${pageSize}`
    ).then(r => r.json()),
  });

  if (isLoading) return <Loading />;

  return (
    <div className="trace-panel">
      <header className="flex justify-between items-center">
        <div>
          <h3 className="font-medium">{data.label}</h3>
          <span className="text-xs text-slate-500">来源: {data.sourceTable}</span>
        </div>
        <button onClick={onClose}>x</button>
      </header>

      {/* SQL查询（折叠） */}
      <details className="mt-2">
        <summary className="text-xs text-slate-400 cursor-pointer">查看SQL</summary>
        <code className="text-xs block mt-1 p-2 bg-slate-50 rounded whitespace-pre-wrap">
          {data.sourceQuery}
        </code>
      </details>

      {/* 数据表格 */}
      <div className="mt-3 overflow-x-auto">
        <table className="w-full text-xs border-collapse">
          <thead>
            <tr className="bg-slate-100">
              {data.columns.map(col => (
                <th key={col.key} className="border px-2 py-1 text-left">{col.label}</th>
              ))}
            </tr>
          </thead>
          <tbody>
            {data.rows.map((row, i) => (
              <tr key={i} className={i % 2 === 0 ? '' : 'bg-slate-50'}>
                {data.columns.map(col => (
                  <td key={col.key} className="border px-2 py-1">{row[col.key]}</td>
                ))}
              </tr>
            ))}
          </tbody>
        </table>
      </div>

      {/* 分页器 */}
      <div className="mt-2 flex items-center justify-between text-xs">
        <span className="text-slate-500">
          共{data.pagination.totalRows}行，第{page}/{data.pagination.totalPages}页
        </span>
        <div className="flex gap-1">
          <button
            disabled={page <= 1}
            onClick={() => setPage(p => p - 1)}
            className="px-2 py-1 border rounded disabled:opacity-30"
          >上一页</button>
          <button
            disabled={page >= data.pagination.totalPages}
            onClick={() => setPage(p => p + 1)}
            className="px-2 py-1 border rounded disabled:opacity-30"
          >下一页</button>
        </div>
      </div>

      {/* 计算公式 */}
      {data.calculations?.length > 0 && (
        <div className="mt-3 border-t pt-2">
          <h4 className="text-xs font-medium text-slate-600">计算公式</h4>
          {data.calculations.map((calc, i) => (
            <div key={i} className="mt-1 p-2 bg-blue-50 rounded text-xs">
              <strong>{calc.metric}</strong>
              <span className="ml-2 font-mono">{calc.formula}</span>
              <div className="text-slate-500 mt-0.5">
                {Object.entries(calc.inputs).map(([k, v]) => `${k}=${v}`).join(', ')}
              </div>
              <div className="text-blue-700 font-medium">= {calc.result}</div>
            </div>
          ))}
        </div>
      )}
    </div>
  );
}
```

```

|  +-------------------------------------+ [D]                 |

|  | 总费用：128,000元                     |   +------------+   |

|  +-----------------------------+ [D]       |            |   |

|  | 曝光 | 500万| 620万| 124%  |           | |指标|.. | |   |
|  | CPM  | 50   | 20.6 | 243%  |           | |曝光|.. | |   |

|  +-----------------------------+           | +--------+ |   |
|                                             |            |   |
|  [D] = 溯源锚点                            | 1/5  < >  |   |
|  hover时显示，点击打开右侧面板               |            |   |
|                                             | 计算公式:  |   |
|                                             | CPM=费用/  |   |
|                                             | 曝光*1000  |   |
|                                             +------------+   |
+--------------------------------------------------------------+
```

- 锚点默认 opacity:0.3，hover 时 opacity:1 + 元素虚线边框高亮
- 面板浮层定位在点击元素右侧（或固定右侧栏）
- 表格每页10行，底部分页器（上一页/下一页）
- 计算公式折叠在表格下方
- 点击其他锚点切换面板内容
- 点击面板外部或X按钮关闭

## 数据量控制

| 策略 | 实现 |
|------|------|
| 前端分页 | 每页10行，API按page/pageSize切片返回 |
| DB存全量 | data_rows存全部行，API负责分页 |
| 列精简 | 只存该段落相关的列，不存全表所有字段 |
| 按需加载 | 锚点点击时才请求API，不预加载 |
| 过期清理 | 报告重新生成时 DELETE WHERE review_config_id |
